# 任务委派指南

**关联**：[M2 管理模式](../methodology/m2-manager-pattern.md) | [都知角色定位](../methodology/coordinator-role.md)

---

## 派任务的标准格式

```
目标：[做什么，一句话]
范围：[哪些文件/目录/资源]
输出：[结果长什么样，多少条]
验收：[什么情况算完成]
```

## 示例

❌ 差的委派：
> 「去看看 bookmark 代码有没有问题」

✅ 好的委派：
> 「审计 components/bookmarks/browser/ 目录，  
>  找所有递归调用无深度限制的函数，  
>  输出：函数名、文件行号、攻击路径（一句话），  
>  验收：覆盖所有反序列化入口」
