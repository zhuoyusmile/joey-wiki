# 工兵陷阱

**日期**：2026-04-09  
**任务**：Chromium bookmark 模块安全审计  
**关联**：[[M2 管理模式]] [[都知角色定位]]

---

## 发生了什么

Joey 让我审计 Chromium bookmark 代码。我自己：

- 一个一个 `grep` 源文件
- 自己写 Python PoC 脚本
- 自己装 `xclip`
- 自己读了十几个 `.cc` 文件
- Joey 发消息，我响应延迟

整个过程持续几个小时，context 被代码细节塞满。

---

## 错在哪里

我把**协调者**的角色和**工兵**的角色混为一谈。

正确做法：

```
小爪：「spawn subagent 审计 bookmark_node_data.cc
       找所有递归反序列化函数，给我漏洞摘要」

小爪等结果，继续和 Joey 对话

subagent 汇报 → 小爪整合 → 发给 Joey
```

---

## 教训

1. **执行工作 = 委派**，不管是写代码还是读文章
2. **读学习材料也是工兵工作**——不要自己一篇一篇翻
3. **判断标准**：做这件事时，还能立刻响应 Joey 吗？不能就委派

---

## 成果（虽然方式错了，结果还可以）

发现了 4 条递归深度无限制漏洞，2 个 patch，1 份完整审计报告。  
→ 见 [[Chromium 审计流程]]
